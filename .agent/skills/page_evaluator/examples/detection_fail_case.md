# 検出失敗事例：石垣島ホテルガイド（英語版）

## 概要
2026年4月の評価において、アフィリエイト設計の重要要素（比較表、マイクロコピー等）を「未実装」と誤判定した事例。

## 発生した問題
- **比較表の未検知**: Next.js + Grid Layout で構築された表を、Markdownパース時にプレーンテキストとして処理し、「表がない」と判断した。
- **装飾付きマイクロコピーの無視**: 絵文字や動的クラス（`animate-pulse`）を含むスパンを見落とした。
- **価格・社会的証明の見落とし**: 特定のCSSティア表示（`$$$`）や、小さなテキストバッジ内のスコア（`8.8`）を無視した。

## 解決策（改善された挙動）
1.  **Markdownのみに頼らない**:
    - `Hotel Comparison Table` という見出しがある場合は、その下の構造を `curl` で精査する。
    - `Check availability` 等の主要CTAがある場所の周辺 HTML を必ず `grep` する。
2.  **キーワードの執念深いスキャン**:
    - `Price Range`, `$$$`, `Review Score`, ` Rating` 等の文字列を RAW HTML 全体で検索する。
3.  **JSON-LDとの突合**:
    - `FAQPage` が JSON-LD にあるのに Markdown に無い場合は、`<details>` タグや特定の JS コンポーネントに隠されていると推測し、ソースを再確認する。

## 教訓
「一見して見当たらない」ことは「未実装」の証明ではない。RITOTABI ページにおいては、特に EN（英語）ページで高度なコンポーネントが多用されるため、**技術的な裏付け（RAW HTMLスキャン）を怠らないこと**。
